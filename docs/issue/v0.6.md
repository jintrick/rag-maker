# Issue v0.6: RAGのプロジェクトへのインストールとパス整合性の確保

## 前提知識
- docs/Architecture/Architecture.md
- discovery.json (マスターツールカタログ)
- <KB_ROOT>/discovery.json (ドキュメントカタログ)

## 経緯
作成したRAGを他のプロジェクトに配置して利用する際のパス不整合問題を解決し、プロジェクトへの「インストール」を容易にする。また、大規模なナレッジベースにおけるメタデータの肥大化問題にも対処する。

## 現状の問題点
1.  **パスの不整合**: RAGをコピー（移動）すると、`discovery.json` 内のファイルパスが機能しなくなる。
2.  **メタデータの肥大化**: ドキュメント数が増え、`discovery.json` が巨大化（1000行超など）すると、LLMのコンテキスト負荷が増大し、処理効率が低下する。

## 解決策の概要
RAGをターゲットプロジェクトへ適切に配置し、メタデータを再構成するツールの提供、および肥大化したメタデータの最適化を行う。

## v0.6 の主なマイルストーン
- **v0.6.1**: `install_kb` ツールの実装。RAGを対象ディレクトリにコピーし、`discovery.json` 内のパスを配置場所に合わせて書き換える。
- **v0.6.2**: 巨大な `discovery.json` への対応。LLMへの負荷軽減のための構造最適化。
