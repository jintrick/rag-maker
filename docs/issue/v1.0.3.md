# Issue v1.0.3 - RAGMaker: True Catalogless Implementation & Directory Structure Cleanup

## 1. 目的 (Objective)
v1.0.0 で提唱された「カタログレス・アーキテクチャ」を、ツールレベルで完全に貫徹する。
現状、ツール側（`http_fetch`, `github_fetch`, `install_kb`, `create_knowledge_base`）に `catalog.json` への依存や生成ロジック、および古いナレッジベース構造（`cache/` サブディレクトリの強制）が残っており、ユーザーが期待する「最小構成の記述的ディレクトリ構造」を阻害している。これらを物理的に排除し、純粋なディレクトリベースの管理を実現する。

## 2. 解決すべき問題点と原因 (Root Cause Analysis)

### 2.1. 執拗な `catalog.json` の生成
- **原因**: 
    - `src/ragmaker/utils.py` の `print_catalog_data` 関数に、`output_dir` が指定された場合に `catalog.json` を保存するロジックが残っている。
    - `http_fetch.py`, `github_fetch.py` の `main()` が、取得完了時に `output_dir` を渡してこの関数を呼び出している。
    - `install_kb.py` の `_install_merged` 関数が、明示的に `catalog.json` を作成・マージしようとする。
- **解決策**: `catalog.json` の書き出しロジックを全ツールから削除する。

### 2.2. `cache/` サブディレクトリの強制
- **原因**:
    - `create_knowledge_base.py` が `cache/` ディレクトリを自動作成する。
    - `install_kb.py` がインストール先を強制的に `cache/` 配下に設定する。
- **解決策**: ユーザーが指定した `{{kb_root}}` 直下にドキュメントを配置できるようにツールを修正する。

### 2.3. 不要な空ディレクトリの混入
- **原因**: 
    - `github_fetch` がリポジトリの構造をそのまま再現するため、`.tmp/cache/` 内に大量のディレクトリ階層が生成される。
    - `rag.md` のトリアージ工程で「ファイル」は削除・移動されるが、空になった「ディレクトリ」が残る。
    - `install_kb` や `move_file` が、これら空ディレクトリも含めて一括コピー/移動してしまう。
- **解決策**: 整理（命名・移動）完了後に、`.tmp/cache/` 内の空ディレクトリを再帰的に削除するステップを導入するか、ツール側で空ディレクトリを無視するようにする。

### 2.4. 廃止済み `/ask` コマンドのテンプレート配布
- **原因**: `create_knowledge_base.py` が、廃止された `ask.toml` を `.gemini/commands/` にコピーしようとする。
- **解決策**: このコピー処理を削除する。

## 3. 完了条件 (Acceptance Criteria)
- [x] `catalog.json` がいかなる工程（取得・整理・配置）においても生成されない。
- [x] ナレッジベースのルートディレクトリ直下に `introduction`, `guide` 等が配置され、余計な `cache/` ディレクトリが作成されない。
- [x] 最終成果物に、整理対象外だった空のディレクトリ（`evals/`, `packages/` 等）が混入していない。
- [x] `.gemini/commands/ask.toml` が新規作成された KB に含まれていない。

## 4. 解決策の詳細 (Solution Detail)

### 4.1. ツール群のカタログ依存排除
- `src/ragmaker/utils.py`: `print_catalog_data` からファイル保存ロジックを削除し、標準出力への JSON 表示のみとする。
- `src/ragmaker/tools/install_kb.py`: `catalog.json` のロード、マージ、保存の全ロジックを削除。
- `src/ragmaker/tools/create_knowledge_base.py`: `cache/` 作成および `ask.toml` コピー処理を削除。

### 4.2. `rag.md` のワークフロー修正
- **ステップ 2.5 (最終整合性チェック)** の拡張: 未整理ファイルの確認に加え、`.tmp/cache/` 内の空ディレクトリを削除する `find -type d -empty -delete` 相当の PowerShell コマンドを実行するよう指示。
- **ステップ 3 (最終配置)** の修正: `{{kb_root}}/cache/` への移動ではなく、`{{kb_root}}` 直下への移動に変更。
